# Green Fire — Signal Saga

## The Archive as Living System

An archive is not a warehouse.

A warehouse stores objects. An archive holds signal — encoded meaning that can be retrieved, recombined, and made useful again in contexts its original authors could not anticipate.

The Green Fire Archive is maintained as a living system: new signal arrives through Threshold, is examined in Workshop, and is remembered in Hearth only when it has earned its place.

---

## On Retrieval

Memory without retrieval is silence.

The purpose of an archive is not merely to hold — it is to make available. A well-maintained archive should answer questions that its creators did not know would be asked.

This requires:
- Consistent structure in how material is recorded
- Meaningful metadata that survives the original context
- A retrieval system that honours the distinction between trusted remembered material and recently-arrived unverified signal

---

## The Heart's Role

The Heart is the resident intelligence of the Ember Node. It does not merely regurgitate stored material.

It synthesises. It connects. It brings remembered works into dialogue with new questions.

When the Heart answers from local knowledge, it should make the provenance of its answer visible. This is Signal Trace — the thread that shows which remembered sources informed the response.

A response without signal trace is a guess. A response with signal trace is a grounded answer.

---

## Cycle of Signal

Signal moves through the node in a cycle:

1. **Threshold** — external material arrives, unverified
2. **Workshop** — material is examined, refined, recombined
3. **Hearth** — refined material is remembered, trusted
4. **Hearth** — the Heart answers from remembered signal
5. **Workshop** — new work is created from retrieved knowledge
6. **Threshold** — refined output exits the node, or returns transformed
